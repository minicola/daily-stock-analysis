# 股票组合管理技能

## 目标
分析用户的股票组合，结合市场行情和技术面，提供加仓/减仓/调仓建议。

## 执行流程

### 1. 收集持仓信息

询问用户提供持仓信息（格式示例）：
```
股票代码, 股票名称, 成本价, 现价, 持仓数量
600559, 老白干酒, 25.24, 13.53, 100
605011, 杭州热电, 32.15, 20.90, 100
```

### 2. 获取实时数据

使用 `data-fetching` skill 中的方法：

```python
from data_provider.base import DataFetcherManager

manager = DataFetcherManager()

# 获取每只股票的实时行情
for code in stock_codes:
    quote = manager.get_realtime_quote(code)

# 获取历史数据（用于技术分析）
df, source = manager.get_daily_data(code, days=60)
```

### 3. 逐只分析

对每只持仓股票进行分析：

**基础指标**（含手续费）：
- 浮动盈亏：需扣除买卖双向手续费后计算真实盈亏（参见下方手续费计算规则）
- 持仓市值：现价 × 持仓数量
- 占组合比例：单只市值 / 总市值
- 如果清仓到手金额：卖出金额 - 佣金 - 印花税 - 过户费

**技术面自动评分**（优先使用）：
```python
from src.services.stock_screener import StockScreener
df, _ = manager.get_daily_data(code, days=30)
score = StockScreener.score_five_dimensions(df, realtime_price=current_price)
# score["total"] 即五维评分, score["breakdown"] 为各维度明细
# score["details"] 含各指标说明（MA排列、量价关系等）
```

**手动降级分析**（API不可用时）：
- 均线位置（MA5/MA10/MA20）
- 趋势判断（上升/下降/横盘）
- 支撑位和压力位
- 成交量变化

**参考 `stock-technical-analysis` 的分析框架**（可选深度分析）

### 4. 组合层面分析

**组合结构**：
- 盈利股 vs 亏损股数量
- 行业分布（避免过度集中）
- 单只股票占比（建议不超过30%）

**风险评估**：
- 深度套牢股（亏损>30%）：优先级最高
- 浅套股（亏损10-30%）：观察为主
- 盈利股：考虑止盈或加仓

### 5. 操作建议

**核心规则：A股最小交易单位为100股（1手），所有买卖建议的数量必须为100的整数倍，严禁出现非100整数倍的数量（如50股、150股、250股等）。**

#### 最小持仓约束

| 当前持仓量 | 可选操作 | 说明 |
|-----------|---------|------|
| **100股（1手）** | **持有** 或 **清仓（卖出100股）** | ⚠️ 不存在"减仓"选项，因为无法卖出不足100股的零头 |
| **200股** | 持有 / 减仓100股 / 清仓200股 | 减仓只能卖出100股 |
| **300股** | 持有 / 减仓100股 / 减仓200股 / 清仓300股 | |
| **N×100股** | 卖出数量必须为100的整数倍，且≤总持仓 | |

#### 操作类型

| 操作类型 | 适用场景 | 建议仓位 | 数量要求 |
|---------|---------|---------|---------|
| **减仓** | 趋势破位、深度亏损、基本面恶化 | 减至30%以下或清仓 | 卖出X00股（必须为100的整数倍） |
| **持有** | 横盘震荡、等待方向明确 | 保持现有仓位 | 不操作 |
| **加仓** | 趋势向上、回调企稳、基本面改善 | 增至20-30% | 买入X00股（必须为100的整数倍） |
| **清仓** | 止损位触发、趋势彻底破坏 | 全部卖出 | 卖出全部持仓 |

#### 数量计算规则

1. **卖出数量** = 向下取整到100的倍数（保守方向：少卖）
2. **买入数量** = 向下取整到100的倍数（保守方向：少买）
3. **计算结果不足100股时，不建议该操作**，改为"持有"或"清仓"二选一

**计算示例**：
- 持仓500股建议减仓60% → 卖出300股（500×60%=300，已是100的倍数）✅
- 持仓300股建议减仓50% → 计算得150股，向下取整为100股 → 卖出100股 ✅
- 持仓100股建议减仓50% → 计算得50股，不足100股 → **改为"持有"或"清仓100股"** ✅（⚠️ 严禁建议"卖出50股"）
- 建议买入金额5000元、股价25元 → 买入200股（5000÷25=200）✅
- 建议买入金额8000元、股价35元 → 买入200股（8000÷35≈228，向下取整到200股）✅
- 建议买入金额2000元、股价35元 → 计算得57股，不足100股 → **不建议买入**，资金不足1手 ✅

## 交易手续费计算规则

所有买卖建议的资金计算必须包含交易手续费，确保用户看到的是**真实交易成本和收益**。

### 费率标准（2024年最新）

| 费项 | 费率 | 收取方向 | 最低收费 |
|------|------|---------|---------|
| 佣金 | 0.025%（万2.5） | 买入 + 卖出（双向） | 5元/笔 |
| 印花税 | 0.05% | 仅卖出（单向） | 无 |
| 过户费 | 0.001% | 买入 + 卖出（双向） | 无 |

### 计算公式

```python
def calc_buy_cost(price: float, quantity: int) -> dict:
    """计算买入总成本"""
    amount = price * quantity
    commission = max(amount * 0.00025, 5)  # 佣金，最低5元
    transfer_fee = amount * 0.00001        # 过户费
    total = amount + commission + transfer_fee
    return {"股票金额": round(amount, 2), "佣金": round(commission, 2),
            "过户费": round(transfer_fee, 2), "买入总成本": round(total, 2)}

def calc_sell_income(price: float, quantity: int) -> dict:
    """计算卖出实际到手"""
    amount = price * quantity
    commission = max(amount * 0.00025, 5)  # 佣金，最低5元
    stamp_tax = amount * 0.0005            # 印花税
    transfer_fee = amount * 0.00001        # 过户费
    net = amount - commission - stamp_tax - transfer_fee
    return {"股票金额": round(amount, 2), "佣金": round(commission, 2),
            "印花税": round(stamp_tax, 2), "过户费": round(transfer_fee, 2),
            "卖出实际到手": round(net, 2)}
```

### 输出要求

- **盈亏计算**：真实盈亏 = 卖出到手 - 买入总成本（非简单的现价-成本价）
- **清仓释放资金**：显示扣除手续费+印花税后的实际到手金额
- **加仓所需资金**：显示含手续费的总成本
- **调仓成本**：卖出手续费 + 买入手续费的总和

## 候选股排除规则（新推荐标的）

在资金再配置时推荐新标的，必须排除以下类型：

| 排除类型 | 识别规则 | 原因 |
|---------|---------|------|
| **创业板** | 代码以 `300xxx` 或 `301xxx` 开头 | 开通门槛高、波动大、20%涨跌幅 |
| **ST/\*ST** | 名称包含 `ST` | 退市风险、基本面恶化 |

**注意**：用户已持有的创业板/ST股票仍需正常分析和提供操作建议，但不作为新配置方向。

## 技能协作

本技能会调用其他技能：

1. **data-fetching**：获取实时行情和历史数据
2. **stock-technical-analysis**：深度技术分析（可选）
3. **market-recommendation**：获取当前市场推荐标的

## 注意事项
- 组合分析基于实时数据，不构成投资建议
- 建议定期（每周）进行组合复盘
- 重大市场变化时及时调整策略
- **所有资金计算必须包含手续费（佣金万2.5+印花税0.05%+过户费0.001%），不得只算裸价×数量**
